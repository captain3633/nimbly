# Nimbly Development Phases

This document explains how Nimbly evolves over time.
Each phase has a clear focus and explicit non-goals.

## Phase 0: Foundation (Complete)
- Backend core
- Receipt ingestion
- Factual insights
- Trust-first Savvy behavior

Non-goals:
- UI
- Predictions
- Optimization

## Phase 1: Usable Interface (In Progress)
- Mobile and web shells
- Auth flow
- Receipt upload and review
- Insight feed (read-only)

Focus:
- UX clarity
- Flow correctness
- Calm presentation

Non-goals:
- Advanced visuals
- Personalization
- Recommendations

## Phase 2: Understanding (Planned)
- Improved OCR
- Stronger parsing confidence
- More insight types
- Better explanations

## Phase 3: Learning
- Longitudinal patterns
- User baselines
- Early predictive signals (transparent)

## Phase 4: Guidance
- Gentle suggestions
- Timing nudges
- Still no guarantees
